Angular Overview

See also Angular Fundamentals https://angular.io/guide/architecture
  • Standalone Angular applications are complex.
  • The Angular CLI hides this complexity so you can focus on functionality.
  • It encourages you to use best practices according to the Angular Style Guide https://angular.io/guide/styleguide.
  • These are guidelines that make it easier to maintain the application as it grows.
  • TypeScript is a superset of JavaScript developed by Microsoft.
  • Brings in useful features of current and future versions of JavaScript, which are not yet supported in web browsers.
  • It compiles to JavaScript that can be run in a web browser.
  • Embraced by Google for the development of Angular and is the recommended language for Angular development.
  • Angular has a modular architecture.
  • Angular modules group together sets of functionality.
  • There is a single root module (usually called AppModule) that is run (bootstrapped) when the application starts.
  • Angular modules are decorated with @NgModule where other modules are imported and application components are declared.
  • Components are the basic building blocks of Angular functionality and user-interface.
  • Follow similar approach to other JavaScript frameworks and modelled on the up-coming Web Components standard.
  • Angular components are decorated with @NgComponent where the component selector, html template and css files are declared.
  • Component approach is a UI design philosophy.
  • Components are not created for reusability. They are for created simplicity to break the application down into understandable parts.
  • Features in the application should be composed of a component tree that reflects the UI structure.
