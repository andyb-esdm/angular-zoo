Angular Overview

See also Angular Fundamentals https://angular.io/guide/architecture
  • An SPA is a web application that dynamically rewrites the current page rather than loading entirely new pages from a server.
  • It avoids interruption between successive pages, making it behave more like a desktop application.
  • Either all the necessary HTML, JavaScript, and CSS is retrieved with a single page load, or it's dynamically loaded and added to the page in response to user actions.
  • The page never reloads nor does control transfer to another page. However, the HTML5 History API can be used to provide the perception of separate pages in the application.
  • Interaction with an SPA usually involves dynamic communication with a web API behind the scenes.
  • Modern JavaScript frameworks such as React, Vue, and Angular are designed to make creating SPAs much easier than writing in pure JavaScript or with libraries like jQuery.
  • Angular is written by Google and is designed to make it easier to build web applications.
  • It's called Angular because HTML has angular brackets and Angular manipulates and adds to HTML elements as directives and components
  • Angular (v2+) is not the same as AngularJS (v1) though it shares some similarities. Angular is a new framework that's just about one year old.
  • Angular is heavily opinionated and a comprehensive framework. It integrates what its authors consider to be design best practices when building large, enterprise applications.
  • More recently Angular is also being targeted at mobile (through Ionic and NativeScript). And desktop (through Node.js and Electron).
  • Google have committed to supporting Angular and use it for some of its largest applications (e.g. AdWords)
  • Standalone Angular applications are complex.
  • The Angular CLI hides this complexity so you can focus on functionality.
  • It encourages you to use best practices according to the Angular Style Guide https://angular.io/guide/styleguide.
  • These are guidelines that make it easier to maintain the application as it grows.
  • TypeScript is a superset of JavaScript developed by Microsoft.
  • Brings in useful features of current and future versions of JavaScript, which are not yet supported in web browsers.
  • It compiles to JavaScript that can be run in a web browser.
  • Embraced by Google for the development of Angular and is the recommended language for Angular development.
  • The Angular framework and Angular applications you write have a modular architecture.
  • Angular modules group together related sets of functionality.
  • There is a single root module (usually called AppModule) that is run (bootstrapped) when the application starts.
  • Angular modules are decorated with @NgModule where other modules are imported and application components are declared.
  • Components are the basic building blocks of Angular functionality and user-interface.
  • They belong to Angular modules.
  • Follow similar approach to other JavaScript frameworks and modelled on the up-coming Web Components standard.
  • Angular components are decorated with @NgComponent where the component selector, html template and css files are declared.
  • Component approach is a UI design philosophy.
  • Components are not created for reusability. They are created for clarity to break the application down into understandable parts.
  • Features in the application should be composed of a component tree that reflects the UI structure.
